# Data Files & Storage

> Where Orbiton stores everything on your computer, what it saves, and how to manage it.

---

## Quick Reference

| File / Directory | Purpose | Location | Can Delete? |
|-----------------|---------|----------|-------------|
| `~/.neuro_link_memory.json` | Your personal facts (name, likes, etc.) | `C:\Users\<You>\` on Windows | Yes — Orbiton forgets you |
| `~/.neuro_link_intel/` | Custom knowledge databases | `C:\Users\<You>\` on Windows | Yes — loses custom intel |
| `~/.neuro_link_wiki_cache/` | Cached Wikipedia articles | `C:\Users\<You>\` on Windows | Yes — just re-downloads |
| `%TEMP%/neuro_link_tts.mp3` | Temporary TTS audio files | `C:\Users\<You>\AppData\Local\Temp\` | Yes — auto-deleted anyway |

---

## Detailed Breakdown

### 1. User Memory — `~/.neuro_link_memory.json`

**What it stores:** Facts about you that Orbiton learns from conversation.

**How it learns:** When you say things like:
- "I am Ayman" → saves `{"name": "Ayman"}`
- "I like aviation" → saves `{"likes": "aviation"}`
- "I hate waiting" → saves `{"dislikes": "waiting"}`
- "I live in Doha" → saves `{"location": "Doha"}`
- "My birthday is January 1st" → saves `{"birthday": "January 1st"}`
- "I study computer science" → saves `{"study": "computer science"}`
- "My goal is to build a company" → saves `{"goals": "to build a company"}`

**Full list of keys:**

| Key | Trigger Phrases | Example Value |
|-----|----------------|---------------|
| `name` | "I am...", "I'm...", "My name is...", "Call me..." | `"Ayman Haidry"` |
| `likes` | "I like...", "I love...", "I enjoy..." | `"aviation, coding"` |
| `dislikes` | "I hate...", "I dislike..." | `"waiting, meetings"` |
| `job` | "I work as...", "I am a...", "My job is..." | `"student"` |
| `location` | "I live in...", "I am from...", "My city is..." | `"Doha"` |
| `birthday` | "My birthday is...", "I was born on..." | `"January 1st"` |
| `study` | "I study...", "My major is...", "I am studying..." | `"computer science"` |
| `goals` | "My goal is...", "I want to..." | `"build a company"` |

**File format:** Plain JSON. You can open it in any text editor.

**Example content:**
```json
{
  "name": "Ayman Haidry",
  "likes": "aviation, coding, space",
  "location": "Doha",
  "study": "computer science"
}
```

**Location by OS:**

| OS | Full Path |
|----|-----------|
| Windows | `C:\Users\<Username>\.neuro_link_memory.json` |
| macOS | `/Users/<Username>/.neuro_link_memory.json` |
| Linux | `/home/<Username>/.neuro_link_memory.json` |

**Can you delete it?** Yes. Orbiton will say "I do not know anything about you yet" and start learning again.

**Is it private?** Yes. Never uploaded anywhere. Stays on your machine.

---

### 2. Custom Intel — `~/.neuro_link_intel/`

**What it stores:** Custom knowledge databases you or contributors create.

**How it works:**
- Orbiton has built-in knowledge (constellations, moon phases, aviation facts, space facts)
- You can add your own `.json` files here and Orbiton will load them
- Files must be named `intel_<name>.json`

**Example custom intel file:** `intel_cars.json`
```json
{
  "tesla": "Tesla is an electric vehicle company founded by Elon Musk.",
  "ford": "Ford is an American automaker founded by Henry Ford in 1903."
}
```

**Location by OS:**

| OS | Full Path |
|----|-----------|
| Windows | `C:\Users\<Username>\.neuro_link_intel\` |
| macOS | `/Users/<Username>/.neuro_link_intel/` |
| Linux | `/home/<Username>/.neuro_link_intel/` |

**Can you delete it?** Yes. You lose custom knowledge but built-in knowledge remains.

**Is it private?** Yes. Local only.

---

### 3. Wikipedia Cache — `~/.neuro_link_wiki_cache/`

**What it stores:** Cached Wikipedia article summaries to avoid re-downloading.

**How it works:**
- When you ask "tell me about Mars", Orbiton fetches from Wikipedia REST API
- The result is saved as a `.txt` file here
- Next time you ask, it loads from disk instead of the internet

**File format:** Plain text files named `<topic>.txt`

**Example:** `mars.txt` might contain:
```
Mars is the fourth planet from the Sun and the second-smallest planet in the Solar System...
```

**Location by OS:**

| OS | Full Path |
|----|-----------|
| Windows | `C:\Users\<Username>\.neuro_link_wiki_cache\` |
| macOS | `/Users/<Username>/.neuro_link_wiki_cache/` |
| Linux | `/home/<Username>/.neuro_link_wiki_cache/` |

**Can you delete it?** Yes. Orbiton will just re-download articles when asked.

**Is it private?** Yes. Local only. (Note: the original fetch goes to Wikipedia's public API, but the cache is local.)

---

### 4. Temporary TTS Files — `%TEMP%/neuro_link_tts.mp3`

**What it stores:** Temporary audio files generated by Edge TTS before playback.

**How it works:**
- Orbiton generates speech as an MP3 file
- Plays it through your default audio player
- The file is temporary and should be overwritten on next speech

**Location by OS:**

| OS | Full Path |
|----|-----------|
| Windows | `C:\Users\<Username>\AppData\Local\Temp\neuro_link_tts.mp3` |
| macOS | `/tmp/neuro_link_tts.mp3` |
| Linux | `/tmp/neuro_link_tts.mp3` |

**Can you delete it?** Yes. It will be recreated next time Orbiton speaks.

**Is it private?** Yes. Contains only what Orbiton said to you, which you already heard.

## What Orbiton Does NOT Store

| Data | Stored? | Why? |
|------|---------|------|
| Voice recordings | ❌ No | Speech is processed in real-time, never saved |
| Search history | ❌ No | URLs are opened in browser, not logged |
| Location data | ❌ No | Weather city is parsed from speech, not saved |
| Clipboard history | ❌ No | Clipboard is read once per command, not stored |
| Browser history | ❌ No | Orbiton opens Chrome, but does not track what you do |
| Crash logs | ❌ No | Errors print to terminal, not saved to disk |

## How to View Your Data

### Windows (PowerShell)
```powershell
# View your memory
Get-Content $env:USERPROFILE\.neuro_link_memory.json | ConvertFrom-Json

# List custom intel files
Get-ChildItem $env:USERPROFILE\.neuro_link_intel\

# List wiki cache
Get-ChildItem $env:USERPROFILE\.neuro_link_wiki_cache\
```

### macOS / Linux (Terminal)
```bash
# View your memory
cat ~/.neuro_link_memory.json | python -m json.tool

# List custom intel files
ls ~/.neuro_link_intel/

# List wiki cache
ls ~/.neuro_link_wiki_cache/
```

### Python (any OS)
```python
import json
from pathlib import Path

# View memory
with open(Path.home() / ".neuro_link_memory.json") as f:
    print(json.load(f))

# List intel files
intel_dir = Path.home() / ".neuro_link_intel"
for f in intel_dir.glob("*.json"):
    print(f.name)
```

## How to Delete Your Data

### Nuclear option — delete everything
```python
from pathlib import Path
import shutil

# Delete memory
(Path.home() / ".neuro_link_memory.json").unlink(missing_ok=True)

# Delete custom intel
shutil.rmtree(Path.home() / ".neuro_link_intel", ignore_errors=True)

# Delete wiki cache
shutil.rmtree(Path.home() / ".neuro_link_wiki_cache", ignore_errors=True)

print("Orbiton has forgotten everything.")
```

### Selective deletion
```python
from pathlib import Path
import json

# Just delete your name
with open(Path.home() / ".neuro_link_memory.json", "r+") as f:
    data = json.load(f)
    data.pop("name", None)  # Remove just the name
    f.seek(0)
    json.dump(data, f)
    f.truncate()
```

## Privacy FAQ

**Q: Does my data leave my computer?**  
A: No. Memory, intel, and cache are all local files. The only external calls are:
- Edge TTS (Microsoft, for speech generation)
- Wikipedia REST API (for knowledge lookup)
- Google/YouTube/FlightRadar24 (when you open URLs in browser)

**Q: Can someone else read my memory file?**  
A: Only if they have access to your computer user account. The file has the same permissions as your Documents folder.

**Q: Does Orbiton sync between devices?**  
A: No. Each device has its own memory file. Future generations (Odyssey+) may add sync via online accounts.

**Q: What happens if I uninstall Orbiton?**  
A: The data files stay behind. You must delete them manually (see "How to Delete Your Data" above).

**Q: Is the data encrypted?**  
A: No. It's plain JSON. If you need encryption, encrypt your user profile folder at the OS level.

## For Contributors

If you add a new feature that creates files:

1. **Use `Path.home()` as the base directory.** Never hardcode `C:\Users\...`.
2. **Prefix with `.neuro_link_`.** Follow the existing naming convention.
3. **Document it here.** Add a new section to this file.
4. **Make it optional.** If the file is missing, the feature should degrade gracefully.
5. **Respect privacy.** Never store voice recordings, keystrokes, or screen captures.
